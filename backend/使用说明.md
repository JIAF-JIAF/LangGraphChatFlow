# 智能客服 Agent 使用说明

## 快速开始

### 1. 安装依赖

```bash
cd backend
pip install -r requirements.txt
```

### 2. 配置 API Key

**方式一：使用 .env 文件（推荐）**

在项目根目录创建 `.env` 文件：

```env
API_KEY=your_deepseek_api_key
BASE_URL=https://api.deepseek.com/v1
MODEL=deepseek-v4-pro
EMBEDDING_MODEL=deepseek-chat
```

**方式二：使用 config.json 文件**

编辑 `config.json` 文件：

```json
{
  "api_key": "your_actual_api_key_here",
  "base_url": "https://api.deepseek.com/v1",
  "model": "deepseek-v4-pro",
  "embedding_model": "deepseek-chat"
}
```

**配置说明：**
- `api_key`: DeepSeek API Key
- `base_url`: 固定为 `https://api.deepseek.com/v1`
- `model`: 支持 deepseek-v4-pro 等模型
- `embedding_model`: 向量化模型，默认 deepseek-chat

### 3. 准备知识库内容

编辑 `knowledge_base/` 目录下的文件，填入业务知识内容。

格式建议:
- 使用 `##` 分隔不同主题
- 每个主题下用段落详细说明
- 可以包含: 公司介绍、产品信息、常见问题、联系方式等

支持的文件格式：
- `.txt` - 纯文本文件
- `.pdf` - PDF 文档
- `.docx` - Word 文档

### 4. 启动服务

**启动 MCP 服务器（工具服务）:**
```bash
python mcp_module/start.py
```

MCP 服务器运行在: `http://localhost:8080/mcp`

**启动应用服务器:**
```bash
python app.py
```

后端服务运行在: `http://localhost:5000`

## API 接口

### 1. 检查服务状态

**请求:**
```
GET /start
```

**响应示例:**
```json
{
  "status": "ready",
  "message": "客服系统已就绪 (LangChain)",
  "model": "deepseek-v4-pro",
  "knowledge_base": true
}
```

### 2. 发送对话请求

**请求:**
```
POST /chat
Content-Type: application/json

{
  "message": "你们有哪些产品?",
  "session_id": "optional_session_id"
}
```

**请求参数说明:**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | string | 是 | 用户输入的消息内容 |
| session_id | string | 否 | 会话 ID，用于保持对话上下文 |

**响应示例:**
```json
{
  "reply": "我们提供以下产品:\n1. 智能办公系统...\n2. 数据分析平台...\n3. 客户关系管理系统...",
  "tool_calls": [],
  "session_id": "session_id_here",
  "finished": false
}
```

**响应参数说明:**
| 参数 | 类型 | 说明 |
|------|------|------|
| reply | string | AI 生成的回复内容 |
| tool_calls | array | 工具调用记录列表 |
| session_id | string | 当前会话 ID |
| finished | boolean | 对话是否结束 |

## MCP 架构说明

### 架构概述

系统采用 MCP（Model Context Protocol）架构，将工具服务独立部署：

1. **MCP 服务器**: 运行在 `http://localhost:8080/mcp`，负责管理和提供工具服务
2. **应用服务器**: 运行在 `http://localhost:5000`，负责处理对话请求和 AI 推理
3. **工具调用**: 通过 Streamable HTTP 协议进行通信

### 工具注册机制

工具通过 `@register_tool` 装饰器注册到全局注册表：

```python
from mcp_module.tools.registry import register_tool

@register_tool(
    name="get_weather",
    description="查询指定城市的实时天气",
    parameters=[
        {"name": "city", "type": "string", "description": "要查询天气的城市名称", "required": True}
    ],
    return_type="string"
)
def get_weather(city: str) -> str:
    # 工具实现
    pass
```

## 常见问题

### 1. API Key 配置错误

**错误信息:** `Authentication error` 或 `Invalid API key`

**解决方法:**
- 检查 `.env` 或 `config.json` 文件是否正确配置了 `api_key`
- 确认 API Key 有效且未过期

### 2. 知识库向量化失败

**错误信息:** `生成向量失败`

**解决方法:**
- 检查网络连接
- 确认 API Key 有 embedding 模型权限
- 检查 db 目录是否有写入权限

### 3. MCP 服务器启动失败

**错误信息:** `[Errno 10048] 通常每个套接字地址只允许使用一次`

**解决方法:**
- 使用 `taskkill /F /PID <PID>` 终止占用 8080 端口的进程
- 等待端口释放后重新启动

### 4. 工具调用失败

**错误信息:** `Unsupported function` 或 `Error executing tool`

**解决方法:**
- 检查 MCP 服务器是否正常运行
- 确认应用能访问 `http://localhost:8080/mcp`
- 检查工具注册是否正确

### 5. 助手创建失败

**错误信息:** 创建助手时出错

**解决方法:**
- 检查配置文件中的模型名称是否正确
- 确认 API Key 有调用模型的权限

### 6. 请求被限流

**错误信息:** `请求过于频繁，请稍后再试`

**解决方法:**
- 等待一段时间后重试
- 调整限流配置（如需）

## 自定义配置

### 修改系统提示词

编辑 `modules/prompt/__init__.py` 中的 `CUSTOMER_SERVICE_PROMPT_TEMPLATE` 变量

### 自定义 FewShot 示例

编辑 `modules/prompt/__init__.py` 中的 `DEFAULT_FEW_SHOT_EXAMPLES` 列表：

```python
DEFAULT_FEW_SHOT_EXAMPLES = [
    {"user_query": "你好", "assistant_response": "您好！请问有什么可以帮助您的？"},
    {"user_query": "你们有什么产品?", "assistant_response": "我们提供多种优质产品..."},
]
```

或在使用时传入自定义示例：

```python
from modules.prompt import create_few_shot_prompt

prompt = create_few_shot_prompt(examples=[
    {"user_query": "问题", "assistant_response": "回答"}
])
```

### 添加新工具

1. 在 `mcp_module/tools/` 创建插件文件
2. 使用 `@register_tool` 装饰器注册工具
3. 在 `mcp_module/start.py` 中导入工具模块
4. 重启 MCP 服务器

```python
from mcp_module.tools.registry import register_tool

@register_tool(
    name="my_tool",
    description="我的自定义工具",
    parameters=[
        {"name": "param1", "type": "string", "description": "参数说明", "required": True}
    ],
    return_type="string"
)
def my_tool(param1: str) -> str:
    return "工具执行结果"
```

### 更新知识库

1. 在 `knowledge_base/` 目录修改文档
2. 重启服务，系统自动重新向量化

## 日志说明

系统使用统一的日志模块，日志格式如下：

```
2026-05-08 19:29:01 - chartflow - INFO - 日志消息
```

日志级别：
- INFO: 常规操作日志
- DEBUG: 调试信息
- WARNING: 警告信息
- ERROR: 错误信息

工具调用时会输出详细日志：
- 参数传递日志
- 执行过程日志
- 结果返回日志
