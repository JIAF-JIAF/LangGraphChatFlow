# 智能客服 Agent - 项目总结

## 已完成功能

### 1. 核心架构
- Flask Web 服务框架
- AI 服务 API 集成（兼容 OpenAI SDK）
- 模块化设计，易于扩展
- 统一 AI 客户端管理 (AIClient)
- **MCP（Model Context Protocol）架构** - 工具独立部署，支持多 Agent 共享调用
- **LangGraph 架构** - 状态图工作流编排，支持状态持久化检查点

### 2. LangGraph 模块
- **分离图定义与业务逻辑**: 将状态图定义与具体业务逻辑分离
  - `LangGraphAgent` (agent.py): 状态图定义、节点路由、工作流编排
  - `RAGWorkflow` (rag.py): RAG 业务逻辑实现（含查询扩展）
- **状态持久化**: 通过检查点（Checkpoint）机制实现会话状态的持久化存储
- **工作流节点**: router（智能路由）、retrieve（文档检索）、generate（生成回答）、call_model（模型调用）
- **查询扩展**: 使用 LLM 扩展用户查询，生成多个相关查询词提高召回率

### 3. 模块化 RAG 框架
- 文档索引模块 (Indexer) - 支持 Chroma、Milvus
- 文档检索模块 (Retriever) - 支持简单向量检索、重排序检索、过滤检索
- 回答生成模块 (Generator) - 支持 Stuff、Map-Reduce、Refine 策略
- 对话记忆模块 (Memory) - 支持 ConversationMemory
- 路由决策模块 (Router) - 决定是否检索及选择策略（含 LLMRouter）
- 自动检测知识库，向量为空时自动重新向量化
- 首次运行自动向量化原始文本（支持 .txt、.pdf、.docx）
- 基于余弦相似度的内容检索
- Chroma 向量数据库支持（兼容 langchain-chroma）
- 向量化失败安全处理

### 4. AI 助手模块
- AI 助手初始化和管理 (assistant.py) - 旧版实现
- LangGraph Agent - 新版实现
- 自动创建/加载助手
- 会话线程管理
- 对话上下文维护
- 通过 MCP 服务器调用工具

### 5. 工具系统（MCP 架构）
- MCP 工具注册机制（通过 `@register_tool` 装饰器）
- 工具独立部署在 MCP 服务器
- 支持 Streamable HTTP 协议
- 支持多个 MCP 服务器配置
- 天气查询工具 (get_weather)
- 天气预报工具 (get_weather_forecast)
- 表单提交工具 (submit_form)
- 可扩展的工具接口

### 6. FewShot 学习系统
- LengthBasedExampleSelector 动态示例选择
- 基于输入长度自动裁剪示例
- 默认示例对话库 (DEFAULT_FEW_SHOT_EXAMPLES)
- 可自定义的 FewShot 示例配置

### 7. Web API
- GET /start - 服务状态检查
- POST /chat - 对话接口
- 会话管理(session_id)
- 限流支持（Flask-Limiter）
- 错误处理和状态码

### 8. 配置管理
- .env 文件支持
- config.json - 系统配置
- mcp_module/config.py - MCP 服务器配置
- 模块化配置读取
- 支持多个 MCP 服务器配置

### 9. 日志系统
- 统一日志模块 (mcp_module/logger.py)
- 结构化日志输出
- 支持不同日志级别

### 10. 限流模块
- Flask-Limiter 集成
- 可配置的限流规则
- 429 错误处理

### 11. 文档
- README.md - 项目说明
- 启动指南.md - 快速入门
- 后端服务运行流程分析.md - 架构分析

## 核心功能说明

### 工作流程

1. **启动阶段**
   - **启动 MCP 服务器**: `python mcp_module/start.py`
   - 工具通过 `@register_tool` 装饰器自动注册到注册表
   - FastMCP 服务器启动，监听 `http://localhost:8080/mcp`
   - **启动应用服务器**: `python app.py`
   - 加载配置文件（.env 或 config.json）
   - 初始化 AI 客户端 (AIClient)
   - 初始化模块化 RAG 链
   - 构建知识库索引（自动检测并向量化）
   - 从 MCP 服务器获取工具列表
   - 创建/加载 LangGraph Agent
   - 启动 Flask 服务

2. **对话阶段**
   - 接收用户消息
   - 创建/获取会话线程
   - 从 MCP 服务器获取工具列表
   - 调用 LangGraph Agent 处理
   - 状态图执行: START → router → [retrieve → generate] → call_model → END
   - RAG 检索增强（Router 判断 → Retriever 检索 → Generator 生成）
   - 检测工具调用需求
   - 通过 MCP 客户端调用远程工具
   - 返回 AI 回复

3. **工具调用流程**
   - AI 判断是否需要调用工具
   - 通过 MCPToolService 向 MCP 服务器发送 HTTP 请求
   - MCP 服务器执行工具并返回结果
   - 将结果返回 AI
   - AI 生成最终回复

### FewShot 学习流程

1. **初始化阶段**
   - 加载 DEFAULT_FEW_SHOT_EXAMPLES 示例列表
   - 创建 LengthBasedExampleSelector 实例
   - 将示例注入到系统提示模板

2. **对话阶段**
   - 根据输入长度选择合适的示例
   - 将示例拼接到系统提示中
   - 发送给 AI 模型处理

### LangGraph 架构流程

```
┌─────────────────────────────────────────────────────┐
│              LangGraph Agent                        │
│  ┌─────────────────────────────────────────────┐   │
│  │           状态图 (StateGraph)                │   │
│  │  START → router → retrieve → generate → call_model → END │   │
│  └─────────────────────────────────────────────┘   │
│                      ↓                             │
│            ┌─────────────────┐                     │
│            │  Checkpoint     │ ← 状态持久化        │
│            └─────────────────┘                     │
└─────────────────────────────────────────────────────┘
                          ↑
                          │ 调用
                          ↓

│  ┌─────────┐ ┌─────────┐ ┌─────────────┐          │
│  │ Router  │ │Retriever│ │ Generator   │          │
│  └────┬────┘ └────┬────┘ └──────┬──────┘          │
│       │           │             │                  │
│       └───────────┼─────────────┘                  │
│                   ↓                               │
│            ┌─────────────────┐                     │

│            └─────────────────┘                     │
└─────────────────────────────────────────────────────┘
```

### 状态定义

```python
class AgentState(TypedDict):
    query: str                    # 用户查询
    session_id: str               # 会话 ID
    chat_history: List[BaseMessage] # 对话历史（统一管理）
    need_retrieve: bool           # 是否需要检索
    documents: List[Document]     # 检索到的文档
    answer: str                   # 生成的回答
```

### 查询扩展机制

RAGWorkflow.retrieve() 支持查询扩展：
1. 接收用户原始查询
2. 使用 LLM 生成 3-5 个相关查询词
3. 对每个查询词执行检索
4. 合并结果并去重
5. 返回最终检索结果

## 技术栈

- **后端框架**: Flask 3.0.0
- **AI SDK**: OpenAI >= 1.40.0（兼容阿里云百炼）
- **向量化**: text-embedding-v3
- **AI 模型**: qwen-plus / qwen3.6-flash
- **向量数据库**: Chroma >= 0.5.0
- **LangChain**: >= 0.3.0 (Agent、工具框架、ExampleSelector)
- **LangGraph**: >= 0.1.0（状态图工作流框架）
- **MCP**: Model Context Protocol（工具服务协议）
- **限流**: Flask-Limiter 3.5.0
- **HTTP 客户端**: requests 2.31.0
- **数值计算**: numpy 2.4.4

## 自定义扩展

### 修改系统提示词

编辑 `modules/prompt/__init__.py` 中的 `CUSTOMER_SERVICE_PROMPT_TEMPLATE` 变量

### 自定义 FewShot 示例

编辑 `modules/prompt/__init__.py` 中的 `DEFAULT_FEW_SHOT_EXAMPLES` 列表

### 添加新工具

1. 在 `mcp_module/tools/` 创建插件文件
2. 使用 `@register_tool` 装饰器注册工具
3. 在 `mcp_module/start.py` 中导入工具模块
4. 重启 MCP 服务

```python
from mcp_module.tools.registry import register_tool

@register_tool(
    name="my_tool",
    description="我的自定义工具",
    parameters=[
        {"name": "param1", "type": "string", "description": "参数说明", "required": True}
    ],
    return_type="string"
)
def my_tool(param1: str) -> str:
    return "工具执行结果"
```

### 更新知识库

1. 在 `knowledge_base/` 修改文档
2. 重启服务自动重新向量化
