# 数据分析技能

## 基本信息

- **名称**: data_analysis
- **标题**: 数据分析技能
- **版本**: 1.0.0
- **作者**: AI Team

## 技能描述

分析销售数据、生成可视化报告和业务洞察。支持多维度数据查询、趋势分析、异常检测等功能。

## 触发条件

### 关键词触发
- 分析
- 报告
- 数据
- 统计
- 趋势

### 难度等级
3

## 执行流程

### 步骤一：理解需求
- **目标**: 明确用户的数据需求和分析维度
- **操作**: 分析用户查询，识别关键要素
- **输入**: 用户自然语言查询
- **输出**: 结构化的需求描述（时间范围、指标、维度）

### 步骤二：数据获取
- **目标**: 获取相关业务数据
- **操作**: 调用数据查询工具获取原始数据
- **输入**: 时间范围、数据类型
- **输出**: 原始数据集
- **工具**: [get_sales_data, get_user_data]

### 步骤三：数据清洗
- **目标**: 处理数据质量问题
- **操作**: 去除重复数据、处理缺失值、识别异常值
- **输入**: 原始数据集
- **输出**: 清洗后的数据集

### 步骤四：数据分析
- **目标**: 提取关键洞察
- **操作**: 计算指标、识别趋势、对比分析
- **输入**: 清洗后的数据集
- **输出**: 分析结果摘要

### 步骤五：可视化（可选）
- **目标**: 生成可视化图表
- **操作**: 根据分析需求选择合适的图表类型
- **条件**: 用户明确要求或数据适合可视化
- **输入**: 分析结果
- **输出**: 图表链接或图片
- **工具**: [generate_chart]

### 步骤六：报告撰写
- **目标**: 生成专业分析报告
- **操作**: 整合分析结果，按照模板输出报告
- **输入**: 分析结果、图表（如有）
- **输出**: 结构化报告
- **工具**: [export_report]

## 工具依赖

| 工具名称 | 说明 | 必填 |
|----------|------|------|
| get_sales_data | 获取销售数据 | 是 |
| get_user_data | 获取用户数据 | 否 |
| generate_chart | 生成图表 | 否 |
| export_report | 导出报告 | 是 |

## 专业知识

### 相关概念
- **KPI指标**: 关键绩效指标，如销售额、转化率、复购率
- **同比分析**: 与去年同期数据对比
- **环比分析**: 与上一周期数据对比
- **漏斗分析**: 用户转化路径分析

### 图表类型选择
- **折线图**: 展示趋势变化
- **柱状图**: 数据对比分析
- **饼图**: 占比分布展示
- **散点图**: 相关性分析

## 示例

**输入**: 分析一下上个月的销售数据

**输出**: 
```markdown
# 数据分析报告

## 概述
2024年1月销售数据分析

## 核心发现
1. 销售额环比增长15%，达到月度目标的108%
2. 华东区域贡献最大，占总销售额的35%
3. 移动端订单占比首次超过PC端，达到52%

## 详细分析
- 热销产品Top3：产品A、产品B、产品C
- 转化率提升主要来自新用户
- 退货率环比下降2%

## 建议
1. 加大移动端推广投入
2. 优化华东区域供应链
3. 针对热销产品开展促销活动
```
